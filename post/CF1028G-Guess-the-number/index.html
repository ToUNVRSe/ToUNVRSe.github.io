题意

本题为交互题

你要猜一个数 x，你每次可以给出不超过 k = min (x,104)（注意你可能不知道 k 的值，一旦超过就立刻错误）个数，将数轴分成 k + 1 段，交互库会返回 [0,k] 表示 x 在哪一段中或 x 恰好是这些数的一个。

询问次数上限为 5，x ∈ [1,10004205361450474]

思路

首先我们发现 x 的范围非常抽象，而且询问次数只有 5 次，所以可以看出这个界应当是非常紧的。

我们知道如果是二分的话，根据实现不同是有少量常数差距的，所以可以猜想这题不是二分。

让我们继续观察，我们注意到如果我们每次询问的 k 必然是当前 x 的下界，并且除了这个下界以及我们还剩下几次询问以外，似乎没有什么能影响我们的询问。

根据传统套路，设 fi, l 表示我们剩余 i 次询问，x 的下界是 l 所能询问出来的区间能有多长。

为了方便写出数值，以下设当前试图询问的区间的左端点为 L。

考虑转移，根据前面的分析，我们这次询问的 k 必然是 min (l,104)，那么如果 x 落在第 0 个区间内，则应该转移至 fi − 1, l，那么第一个分界点就应该是 fi − 1, l + L。

而如果落在第 1 个区间内，则应该转移至 f[i−1,f[i−1][l]+L+1]，其意义并不复杂，因为我们第一个分界点为 fi − 1, l + L，故而我们能确定 x 的下界为 fi − 1, l + L + 1，于是按照定义转移即可。

而 fi, l 的终值，就是所有我们填上去的区间长度的总和。

我们注意到 l > 104 的值都是没有用的，因为再大并不会对我们的决策产生影响，于是我们令 l ∈ [1,104]。如果转移的时候还剩下 k 没有用完，而分界点已经超过了 104，那么剩下的就全填上 fi − 1, 104 + 1 就可以了。
